БЛОК 5. Формат реализации

1. ВХОД В ИГРУ

Сценарий:

— пользователь нажал “Крутим удачу”
→ запускается звук (вау-начало)
→ открывается Game Screen

2. ВНУТРИ GAME МОДУЛЯ

Архитектура:

GlobalFXLayer (общие эффекты)
GameScreen (сцены)
UILayer (кнопки)
3. ОБЯЗАТЕЛЬНО ПЕРЕНЕСТИ В ИГРУ

С главного экрана:

— AI логотип (анимация, не PNG)
— плавающие слова
— магические круги

👉 это единый слой внутри Game

4. ЭКРАН БАРАБАНА

— книга = фон
— магический круг = центр
— барабан = ВНУТРИ круга

❗ НЕ в книге

— круг увеличить
— кнопки поднять

5. ЭКРАН ПРЕДСКАЗАНИЯ

— появляется внутри книги
— компактный
— ниже центра

— кнопка “Продолжить”:
— всегда активна
— не зависит от звуков

6. ЛОГИКА ИГРЫ (ОСНОВА)

Каждое действие = сценарий:

Крутка:

— звук прокрутки
— ускорение

Финал крутки:

— тарелка
— + эффект открытия
— = открытие предсказания

После:

— 1 случайный смех

7. ЗВУКИ

❗ Только внутри Game логики
❗ Без фоновой музыки

Использовать:

— прокрутка
— тарелка
— 6 вариантов эффектов открытия
— 3+ варианта смеха

8. ПОВТОРЫ

— пользователь нажимает “Продолжить”
→ возврат к барабану
→ та же логика

9. ФИНАЛ ИГРЫ

(внутри Game)

Только здесь:

— фанфары
— аплодисменты
— фейерверк

10. БАЗА СЛОВ (ВНУТРИ GAME)

❗ Это часть игровой логики

Система должна:

— сохранять все добавленные слова
— не перезаписывать
— накапливать

Поведение:

— пользователь добавил слово
→ слово добавилось в базу
→ участвует в следующих прокрутках

Реакции:

— эффекты тоже фиксируются
— используются повторно

👉 Это растущая игровая база

11. ОЧИСТКА

Удалить ТОЛЬКО В GAME:

— PNG AI
— дубли эффектов
— кнопки звука
— фоновую музыку
— лишние подписи
