# Инструкция: как открыть сайт локально и видеть все изменения

Эта инструкция нужна, чтобы вы всегда открывали **актуальную версию** проекта на своем компьютере и не путались, какие правки уже внесены.

## 1) Открыть проект в Cursor

1. Откройте папку проекта:
   `c:\Users\Таня\Desktop\МАМА УДАЛЕНКА\КУРСОР ПРОЕКТЫ\ИТОГ СЛОВАРЬ СЛЭНГА`
2. Убедитесь, что рабочая папка приложения:
   `magic-book-app`

## 2) Запуск сайта локально (режим разработки)

Откройте терминал в Cursor и выполните:

```powershell
cd "c:\Users\Таня\Desktop\МАМА УДАЛЕНКА\КУРСОР ПРОЕКТЫ\ИТОГ СЛОВАРЬ СЛЭНГА\magic-book-app"
npm install
npm run dev
```

После запуска откройте адрес, который покажет Vite (например):

- `http://localhost:5173/`
- `http://localhost:8080/`

Важно: именно здесь вы видите изменения сразу после правок.

## 3) Как проверить, что вы смотрите нужную версию

Перед тестом всегда проверяйте:

```powershell
cd "c:\Users\Таня\Desktop\МАМА УДАЛЕНКА\КУРСОР ПРОЕКТЫ\ИТОГ СЛОВАРЬ СЛЭНГА"
git status
```

Что это дает:

- видно, какие файлы изменены;
- видно, есть ли незакоммиченные правки;
- можно понять, на какой версии вы сейчас тестируете задачу.

## 4) Как сравнить, что именно изменилось

```powershell
cd "c:\Users\Таня\Desktop\МАМА УДАЛЕНКА\КУРСОР ПРОЕКТЫ\ИТОГ СЛОВАРЬ СЛЭНГА"
git diff
```

Если нужны только конкретные файлы:

```powershell
git diff -- "magic-book-app/src/pages/Index.tsx"
git diff -- "magic-book-app/public/slovar/script.js"
git diff -- "magic-book-app/public/slovar/style.css"
```

## 5) Как открывать именно словарь в iframe (экран intro)

В локальном режиме откройте адрес из терминала Vite (например):

- `http://localhost:5173/`
- `http://localhost:8080/`

Это вход через React, где словарь подключается через iframe:

- `src="/slovar/index.html"`

Если открыть напрямую (подставьте тот же порт):

- `http://localhost:5173/slovar/index.html`
- `http://localhost:8080/slovar/index.html`

это будет автономный режим словаря (без React-обвязки).

## 6) Рекомендуемый порядок проверки после каждой задачи

1. Запустить `npm run dev`.
2. Открыть адрес, который выдал Vite (`localhost:5173` или `localhost:8080`).
3. Очистить кеш браузера (или Hard Reload).
4. Проверить сценарий задачи.
5. Посмотреть `git status` и `git diff`.
6. Зафиксировать, какие файлы затронуты.

## 7) Если "не вижу изменения"

Проверьте по шагам:

1. Вы точно в папке `magic-book-app`.
2. Dev-сервер действительно запущен (`npm run dev`).
3. Открыт `localhost`, а не старая вкладка Vercel.
4. Сделан Hard Reload в браузере.
5. В терминале нет ошибок сборки.

## 8) Где смотреть онлайн-версию

Локальная версия (для проверки новых правок) и онлайн-версия (деплой) могут отличаться.

- Локально: адрес из Vite (`http://localhost:5173/` или `http://localhost:8080/`)
- Vercel: `https://itog-slovar-slang.vercel.app/`

Сначала проверяйте локально, потом сравнивайте с Vercel.

## 9) Как работает деплой на Vercel (без CLI)

Важно: у вас может быть не установлен `Vercel CLI` (`vercel` команда в терминале).
Это нормально.

У вас работает автодеплой:

`локальные изменения -> git push -> GitHub -> Vercel автосборка и публикация`

Что это значит на практике:

- вручную запускать `vercel --prod` не обязательно;
- после `push` Vercel сам обновляет сайт;
- если хотите проверить результат, открывайте:
  - `https://itog-slovar-slang.vercel.app/`

## 10) Где сохраняются все внесенные слова и скрины

Основная рабочая база словаря хранится в браузере (localStorage) по ключам:

- `magic-book-entries` — основная база (слова + описания + реакции + скрины).
- `magic-book-entries-lite-backup` — облегченный backup (слова/описания/реакции, без base64-скринов).
- `magic-book-entries-max-count` — контроль максимального количества слов.

Дополнительно в проекте есть файл восстановления:

- `magic-book-app/public/recovered-entries.json` — офлайн-восстановленный список (если удалось извлечь из браузерной памяти).

## 11) Правило сохранности данных (ОБЯЗАТЕЛЬНО)

- Внесенные слова и скрины НЕ чистить.
- Внесенные слова и скрины НЕ править массово.
- Режим работы только: добавить -> сохранить -> продолжить с последнего сохраненного номера.
- Нельзя возвращать словарь к началу, если в базе уже есть пользовательские записи.

## 12) Как восстановить после потери

1. Открыть:
   - `http://localhost:5501/?entry=slovar&screen=form`
2. Проверить счетчик `Всего слов: N`.
3. Если счетчик просел:
   - использовать `magic-book-entries-lite-backup`;
   - если доступен, поднимать из `recovered-entries.json`.
4. После восстановления сразу проверить переход в чтение:
   - `http://localhost:5501/?entry=slovar&screen=reading`
5. После подтверждения сделать фиксацию в `audit_plan.md` и коммит.

1. Открыть:
   - `http://localhost:5501/?entry=slovar&screen=form`

- Можно чистить только: `Cached images and files` (кэш картинок/файлов).

  - `Cookies`,
  - `Site data`,
  - `Local storage`.

1. Снова открыть:
   - `http://localhost:5501/?entry=slovar&screen=form`
2. Нажать `Ctrl+F5`.
3. Сверить `Всего слов: N` с числом до очистки.
